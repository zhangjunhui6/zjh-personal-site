# 个人网站设计文档

日期：2026-05-12  
状态：草案，待用户确认  
方向：Warm Studio / 长期记录型个人空间

## 1. 项目目标

这个个人网站不是传统简历页，也不是纯博客模板。它的目标是成为一个可以长期生长的个人空间，用来沉淀文章、生活记录、项目实验和阶段性的想法。

第一版优先做到：

- 看起来温暖、松弛、有个人气质，同时保持专业和克制。
- 内容写作成本低，使用 Markdown/MDX 管理。
- 免费、开源、低维护，适合部署在 Cloudflare Pages。
- 技术结构不要过重，但为搜索、评论、CMS 和中英双语预留扩展空间。

## 2. 第一版范围

第一版包含 5 个主页面：

- `Home`：个人空间入口，展示最近记录、精选内容、项目和生活切片。
- `Notes`：文章、技术笔记、读书想法、长期思考。
- `Journal`：生活记录、旅行、影像、碎片化观察。
- `Projects`：小工具、实验项目、作品记录。
- `About`：关于个人、长期关注方向、联系方式。

第一版暂不做：

- 网页后台编辑。
- 评论系统。
- 站内搜索。
- 登录、数据库、服务端 API。
- 完整英文内容。

这些能力只保留结构入口，等内容稳定后再接入。

## 3. 首页设计

首页要像“一间温暖的个人工作室”，而不是营销落地页。

首屏包含：

- 一句自然的自我介绍。
- 当前关注方向，例如技术、写作、生活观察、独立项目。
- 近期状态或一小段近况文字。

首屏之后包含：

- 最近写下的内容：来自 `Notes`。
- 最近生活记录：来自 `Journal`。
- 正在做或做过的项目：来自 `Projects`。
- 一个轻量的 About 入口。

首页不堆大段介绍，不做强销售感 CTA。核心体验是：别人进来能感受到这是一个真实的人正在持续记录和创造。

## 4. 内容模型

使用 Astro Content Collections 管理内容。

### Notes

用于较完整的文章和想法。

字段：

- `title`
- `description`
- `date`
- `updated`
- `tags`
- `lang`
- `draft`

### Journal

用于生活记录、照片记录、短篇观察。

字段：

- `title`
- `description`
- `date`
- `mood`
- `location`
- `images`
- `tags`
- `lang`
- `draft`

### Projects

用于项目、工具和实验。

字段：

- `title`
- `description`
- `date`
- `status`
- `stack`
- `links`
- `cover`
- `featured`
- `lang`

## 5. 技术方案

采用方案 A：

- 框架：Astro + TypeScript
- 内容：Markdown/MDX
- 样式：Tailwind CSS
- 内容管理：Astro Content Collections
- 部署：Cloudflare Pages
- 代码托管：GitHub
- 开源协议：MIT

构建方式：

- 构建命令：`npm run build`
- 输出目录：`dist`
- 本地开发：`npm run dev`

第一版需要内置：

- 基础 SEO metadata。
- sitemap。
- RSS。
- 404 页面。
- 响应式布局。
- 中文排版优化。

## 6. 国际化策略

第一版只发布中文内容，但代码结构预留英文版本。

约定：

- 当前默认语言为 `zh`。
- 内容字段保留 `lang`。
- 后续英文内容可使用 `/en` 路由扩展。
- 导航、站点配置和内容集合不要写死为只能中文。

第一版不需要完整语言切换器，避免增加无内容页面。

## 7. 视觉风格

方向为 Warm Studio。

关键词：

- 温暖
- 松弛
- 个人化
- 克制
- 可长期阅读
- 有生活感但不随意

设计原则：

- 排版优先，避免花哨动画。
- 使用真实内容、项目截图或生活图像，而不是空洞装饰。
- 页面留白舒适，但信息密度不要太低。
- 卡片只用于内容条目，不把整个页面做成卡片堆叠。
- 色彩避免单一色系，使用温暖中性色搭配少量清爽强调色。

## 8. 后续扩展

内容达到一定规模后，可以逐步加入：

- Pagefind：静态站内搜索。
- giscus：基于 GitHub Discussions 的评论。
- Decap CMS：网页后台编辑 Markdown。
- Cloudflare Web Analytics：轻量统计。
- 自定义域名。

这些扩展不进入第一版实现范围，但第一版目录和组件设计需要避免阻碍后续接入。

## 9. 验收标准

第一版完成时应满足：

- 可以本地启动并浏览所有主页面。
- 首页能够表达“长期记录型个人空间”的定位。
- `Notes`、`Journal`、`Projects` 都有示例内容和列表页。
- 内容集合 schema 能校验 Markdown frontmatter。
- 构建命令通过。
- 页面在桌面和手机宽度下都可读、无明显溢出。
- 项目可以连接 Cloudflare Pages 部署。
