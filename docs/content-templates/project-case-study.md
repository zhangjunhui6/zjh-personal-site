# 项目复盘模板

适用于 `src/content/projects`。

```md
---
title: 项目标题：说明项目价值
description: 用一句话说明这个项目解决了什么问题，以及为什么值得展示。
date: YYYY-MM-DD
status: active
stack: [Astro, TypeScript]
links: []
featured: false
lang: zh
draft: true
---

开头用 1-2 段说明项目是什么、为什么做、当前做到什么程度。

## 为什么做这个项目

说明动机。最好从问题开始，而不是从技术栈开始。

## 项目目标

- 目标一。
- 目标二。
- 目标三。

## 技术栈

- 技术一：为什么选它。
- 技术二：它负责哪一层。
- 技术三：它解决什么问题。

## 整体架构

```text
输入或内容
  -> 处理层
  -> 展示或发布
```

说明数据怎么流动、模块怎么协作。

## 核心功能

### 功能一

说明用户怎么用，以及背后怎么实现。

### 功能二

继续说明。

## 关键取舍

写清楚没有做什么，以及为什么没有做。

- 没做 A，因为当前阶段收益不够。
- 没做 B，因为维护成本会变高。
- 先做 C，因为它直接改善核心体验。

## 当前状态

说明项目现在是否可用、还有哪些限制。

## 下一步

- 下一步一。
- 下一步二。
- 下一步三。
```
