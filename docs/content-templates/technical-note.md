# 技术文章模板

适用于 `src/content/notes`。

```md
---
title: 标题：用一句话说明文章判断
description: 说明这篇文章解决什么问题，以及读者能获得什么。
date: YYYY-MM-DD
tags: [开发流程, 工具]
lang: zh
draft: true
cover:
media: []
---

开头先说明背景、问题和文章边界。不要一上来堆命令，先告诉读者为什么这件事重要。

## 这篇文章解决什么问题

用 2-3 段说明场景。

- 读者现在可能遇到什么。
- 这篇文章会讲什么。
- 这篇文章不讲什么。

## 核心概念

### 概念一

说明它是什么、为什么重要、和其他概念的关系。

### 概念二

继续拆清楚。

## 常用命令和场景

### 场景一

```bash
command --example
```

解释这条命令什么时候用，参数分别代表什么。

### 场景二

```bash
command --another-example
```

## 工作中怎么用好

写实际经验、团队协作、日常习惯和取舍。

## 常见坑

- 坑一：说明问题和避免方式。
- 坑二：说明问题和避免方式。
- 坑三：说明问题和避免方式。

## 我会保留的一组命令或清单

```bash

```

## 最后

用一段话收束：这项技术真正解决的是什么问题。

## 参考资料

- [官方文档](https://example.com)
