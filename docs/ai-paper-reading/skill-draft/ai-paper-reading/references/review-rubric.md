# Review Rubric

Score each category 0-3.

| Category | 0 | 1 | 2 | 3 |
| --- | --- | --- | --- | --- |
| Source anchors | No figure/table refs | Generic refs | Key refs named | Reading order and purpose clear |
| Method understanding | Abstract repeat | Module summary | Design tradeoffs | Alternatives and failure points |
| Formula/objective | Missing | Generic | Main formula aligned | Variables and implementation mapping |
| Data | Names only | Scale only | Source and distribution | Explains data limits and bias |
| Evaluation | Scores only | Metrics named | Protocol described | Can/cannot prove claims explained |
| Domain contract | Missing | Scattered | Main interfaces clear | Contracts tied to failure risks |
| Reproducibility | Missing | Commands only | Environment boundary | Compute, data, outputs, failures |
| Source quality | No sources | Secondary sources | Paper/official sources | Paper, docs, code cross-checked |

Maximum score: 24.

- 0-11: draft notes, not publishable.
- 12-17: readable but not expert-level.
- 18-21: solid technical article.
- 22-24: high-quality paper reading.

Severity:

- **P0**: factually wrong, misleading, unsafe, or overclaims beyond evidence.
- **P1**: important professional gap: missing contract, formula, evaluation protocol, data limits, or primary source.
- **P2**: structure, clarity, wording, or visual improvement.

Review output:

```text

```
